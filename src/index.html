Index

  1. In or out the viewport

    A very basic example of the IntersectionObserver. It will show a feedback message when a target element enters or exits the viewport.

  2. How much is visible?

    Using the threshold option it is possible to specify at which ratios a notification should be given.

  3. How much is visible: oversized edition

    When the observed target is bigger than the root element, the IntersectionObserver will stop notifying when the intersection ratio stops changing.

  4. How much is visible: horizontal scroll edition

    The IntersectionObserver also works when used on a horizontal scroll page.

  5. Modify the intersection area

    Using the rootMargin option it is possible to shrink/grow the area of the root element. This examples shrinks the root by 80 pixels.

  6. Showing the active chapter

    Example that shows how to modify the root margin so elements that pass the middle of page get notified.

  7. Extending the intersection area

    In this example the root is expanded so elements trigger a notification as they're about to enter the viewport. This can be used for instance to lazy load images as they're about to become visible.

  8. Alternate root element

    The root option allows to specify a different element than the viewport to act as the root element.

  9. Initialize modules JIT

    By extending the margin around the viewport and some JS it is possible to initialize JS functionality as the element is about to come into view.
